Название сайта: Cactus Schems

Краткое описание сайта: Сайт с системой аккаунтов и рекламой, предназначенный для публикации схематик Create.

Основная цель сайта: Предоставить пользователям удобный интерфейс для просмотра, фильтрации и добавления схем с мода Create для игры Minecraft с подробными описаниями, изображениями, ресурсами и ссылками на скачивание.

Основные страницы (Кратко):
    index: Главная страница.
    user: Страница конкретного пользователя.
    search: Страница с сеткой схем, фильтрами.
    schematic: Страница конкретной схемы.
    new-schem: Страница для создания новой схематики.

Более подробное описание страниц:
    index: Главная страница сайта, где написано название сайта, его краткое описание, лого, баннер и многое иное, также должны присутствовать кнопки для перехода на страницу поиска схем, создания схем, регистрации и входа пользователей
    user: Страница определённого пользователя, отображается аватарка пользователя, имя, последний заход, сколько прошло времени с создания аккаунта, его описание (пользователь может сам его редактировать, но изначально там - Я новый пользователь!), его опубликованные схемы, возле имя также должен быть указан статус, по умолчанию его нет.
    search: Страница с сеткой схем (в которых есть изображение (выбранное пользователей), название, краткое описание, теги, автор и кнопка открыть) и фильтром тегов (если нажать один раз то тег включается, второй раз исключается, третий раз просто становится не активным)
    schematic: Страница определённой схематики, в ней, название, подробное описание, теги, масштаб (опционально для публикующего), автор, изображения, видео (опционально для публикующего), требуемые ресурсы, требуемые моды, ссылки на скачивания
    new-schem: Страница для создания новых схематик, среди вариантов, которые можно ввести Название, подробное описание, теги (выбираются из списка, либо можно написать новый), масштаб (опционально), изображения, видео (опционально), требуемые ресурсы (выбираются из списка, либо можно написать тех которых нет (к опр. предметам привязываеся опр. иконка)), требуемые моды (выбираются из списка, либо можно написать тех которых нет (к опр. предметам привязываеся опр. иконка)), файлы (ссылки для скачивания (разрешено лишь .nbt и .txt))

У сайта должна присутствовать система аккаунтов. 
Насчёт new-schem, в разделы теги, изображения, видео, требуемые ресурсы, требуемые моды и файлы должна быть кнока добавить новый тег, изображение и т.д и лишь при нажатии на это можно было это добавить.
